# ResearchRenamer 用户手册

## 1. 软件概述

### 1.1 软件名称
ResearchRenamer - 科研成果文件智能编号与命名系统

### 1.2 软件版本
V1.0.0

### 1.3 软件简介
ResearchRenamer 是一款专门为科研人员设计的文件管理工具，用于批量重命名和编号科研成果文件（如论文、报告、图片等）。该软件能够根据文件的发布日期或现有编号进行智能排序和重命名，支持多种编号格式，帮助科研人员更有效地管理和组织研究成果。

### 1.4 软件功能
- 批量上传科研文件（支持 PDF、DOCX、JPG、PNG 等格式）
- 根据发布日期自动排序
- 支持两种编号模式：顺序编号和偏移编号
- 多种编号格式：括号、点号、下划线、连字符、自定义前缀
- 手动调整发布日期
- 一键导出重命名后的文件为 ZIP 压缩包

## 2. 系统要求

### 2.1 硬件要求
- 处理器：Intel Core i3 或同等性能处理器
- 内存：4GB 及以上
- 存储空间：100MB 及以上可用空间
- 显示器：1024x768 及以上分辨率

### 2.2 软件要求
- 操作系统：Windows 10/11、macOS 10.15+、Linux
- 浏览器：Chrome 90+、Firefox 88+、Safari 14+、Edge 90+
- Node.js：16.0.0 及以上版本（仅开发和部署时需要）

## 3. 安装与部署

### 3.1 本地开发环境
1. **安装 Node.js**
   - 从 [Node.js 官网](https://nodejs.org/) 下载并安装适合您操作系统的 Node.js 版本
   - 验证安装：在命令行中运行 `node -v` 和 `npm -v` 确认版本

2. **克隆项目**
   - 从代码仓库克隆项目到本地
   - 进入项目目录：`cd research-renamer`

3. **安装依赖**
   - 运行命令：`npm install`

4. **启动开发服务器**
   - 运行命令：`npm run dev`
   - 浏览器会自动打开应用，默认地址为 http://localhost:3000

### 3.2 生产部署
1. **构建生产版本**
   - 运行命令：`npm run build`
   - 构建产物会生成在 `dist` 目录

2. **部署到服务器**
   - 将 `dist` 目录中的所有文件部署到您的 Web 服务器
   - 确保服务器支持静态文件服务

## 4. 使用指南

### 4.1 基本操作流程

#### 4.1.1 上传文件
1. 在应用首页，点击或拖拽文件到上传区域
2. 支持同时上传多个文件（PDF、DOCX、JPG、PNG 等格式）
3. 上传后，系统会自动提取文件的发布日期和现有编号

#### 4.1.2 配置编号规则
1. 在左侧工具栏选择编号模式：
   - **时间排序重排**：根据文件日期排序，重新生成连续编号
   - **整体平移跳号**：保留原有编号间隔，仅对数字进行加减运算
2. 设置编号参数：
   - 起始编号（时间排序模式）
   - 编号增减量（平移跳号模式）
   - 目标编号样式（方括号、点号、下划线、连字符、自定义）
   - 连接符
   - 最小位数（数字填充）

#### 4.1.3 预览与调整
1. 在文件列表中查看预览的新文件名
2. 点击日期输入框可手动调整文件的发布日期
3. 点击排序按钮切换排序方式：最新优先、最旧优先、文件名、原有编号

#### 4.1.4 导出文件
1. 配置完成后，点击右上角的"导出重命名文件 (ZIP)"按钮
2. 系统会生成一个包含所有重命名文件的 ZIP 压缩包
3. 下载并解压 ZIP 文件，即可得到重命名后的文件

### 4.2 功能详解

#### 4.2.1 时间排序重排模式
- **功能**：根据文件的发布日期排序，从指定的起始编号开始重新生成连续编号
- **适用场景**：适用于需要按时间顺序重新编号的情况，如整理年度论文

#### 4.2.2 整体平移跳号模式
- **功能**：保留原有编号的间隔，仅对数字进行加减运算
- **适用场景**：适用于在现有编号基础上批量调整编号的情况，如插入新文件后调整后续编号

#### 4.2.3 编号格式选项
- **方括号**：[01] 文件名
- **点号**：01. 文件名
- **下划线**：01_文件名
- **连字符**：01-文件名
- **自定义**：可自定义前缀，如 P-01 文件名

#### 4.2.4 日期提取功能
- 系统会自动从文件名中提取发布日期
- 支持 YYYY-MM-DD 和 YYYY 格式的日期识别
- 若未识别到日期，默认使用当前日期
- 可手动修改日期以调整排序

## 5. 常见问题与解决方案

### 5.1 文件上传问题
- **问题**：无法上传文件
  **解决方案**：检查文件格式是否支持（PDF、DOCX、JPG、PNG），确保文件大小在浏览器允许范围内

### 5.2 日期识别问题
- **问题**：系统未正确识别文件日期
  **解决方案**：手动点击日期输入框修改日期

### 5.3 编号格式问题
- **问题**：编号格式不符合预期
  **解决方案**：在左侧工具栏调整编号格式和参数

### 5.4 导出问题
- **问题**：导出失败
  **解决方案**：检查文件数量是否过多，尝试分批处理
